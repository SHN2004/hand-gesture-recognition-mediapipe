# Two-Hand Sequence Only - Implementation Plan

This plan is now scoped to **sequence two-hand recognition only**.

Out of scope for this plan:

- Static single-hand classifier flow
- Static two-hand classifier flow
- Finger-point-history classifier flow

## Target Flow (Aligned With SentiSign Style)

Adopt a class-key-based training lifecycle for sequence two-hand signs:

1. Use **class keys** (e.g. `CUSTOM_HELLO`) instead of numeric-only IDs for dataset identity.
2. Keep a **`label_map.json`** for this model family.
3. Store samples in **per-class CSV files** (`.../raw/<CLASS>.csv`) instead of one merged CSV.
4. Provide `uv` CLI commands for:
   - `add-sign` (create class key, collect samples)
   - `retrain` (rebuild label map, train, export)
5. Inference reads `idx_to_label` from `label_map.json`, then resolves display words via:
   - explicit `CLASS_TO_WORD` mapping
   - fallback from `CUSTOM_*` -> human-readable text.

## Model Family Boundary (Important)

Even though the repository has other recognizers, this implementation uses a **separate model family** and must not mix feature shapes.

Use only:

- `model/two_hand_sequence_classifier/*`

Do not share datasets/models/label maps with:

- `model/keypoint_classifier/*`
- `model/two_hand_keypoint_classifier/*`
- `model/point_history_classifier/*`

## Data Contract

### Per-frame feature vector

- Base feature: 84 floats (21 landmarks x 2 coords x 2 hands)
- Recommended: append presence bits (`left_present`, `right_present`) -> 86 dims

### Sequence window

- `WINDOW = 30` frames (tunable)
- One training sample = flattened `WINDOW * DIMS`

### Per-class CSV format

Each `raw/<CLASS>.csv` row:

- Columns: flattened sequence features only
- Class is encoded by filename (`<CLASS>.csv`), not by a leading numeric column

Rationale:

- Same pattern as SentiSign: class identity comes from per-class files.

## Naming Rules

### Class key

- Default custom key format: `CUSTOM_<UPPER_SNAKE_CASE_WORDS>`
- Example: word `thank you` -> `CUSTOM_THANK_YOU`

### Word mapping

Maintain `class_to_word.json` for explicit mapping, e.g.:

```json
{
  "CUSTOM_HELLO": "HELLO",
  "CUSTOM_THANK_YOU": "THANK YOU"
}
```

### Inference fallback

If class key not in `class_to_word.json`:

- If key starts with `CUSTOM_`, derive word by removing prefix and replacing `_` with space.
- Otherwise use class key as display text.

## label_map.json Contract

Generated by retrain, owned by training pipeline:

```json
{
  "classes": ["CUSTOM_HELLO", "CUSTOM_THANK_YOU"],
  "label_to_idx": {"CUSTOM_HELLO": 0, "CUSTOM_THANK_YOU": 1},
  "idx_to_label": {"0": "CUSTOM_HELLO", "1": "CUSTOM_THANK_YOU"},
  "input_shape": [30, 86]
}
```

Notes:

- Index assignment comes from deterministic class list ordering (e.g., sorted filenames).
- `input_shape` must match model input exactly.

## uv CLI Flow

All operations use `uv run`.

### Add sign

Example:

```bash
uv run python manage_two_hand_sequence_signs.py add-sign --word "hello" --samples-target 120
```

Behavior:

1. Compute class key (`CUSTOM_HELLO`).
2. Run collection loop (or call collector module) to append sequence samples to:
   - `model/two_hand_sequence_classifier/raw/CUSTOM_HELLO.csv`
3. Update `class_to_word.json` if needed.

### Retrain

Example:

```bash
uv run python manage_two_hand_sequence_signs.py retrain
```

Behavior:

1. Load all `raw/*.csv` files.
2. Build class list from filenames.
3. Rebuild and write `label_map.json`.
4. Train sequence model (`LSTM` default).
5. Export:
   - `sequence_classifier.keras`
   - `sequence_classifier.tflite`

## Inference Flow

`two_hand_sequence_classifier.py` + app integration should:

1. Load `label_map.json` + optional `class_to_word.json`.
2. Run TFLite model on `(WINDOW, DIMS)` features.
3. Convert index -> class key via `idx_to_label`.
4. Resolve display word via `class_to_word` or `CUSTOM_*` fallback.

## Critical Caveats

1. Hand slot stability is mandatory.
   Temporal left/right assignment must remain consistent across frames.
2. Feature shape must stay fixed.
   Changing `WINDOW` or `DIMS` requires retrain and `label_map.json` update.
3. Do not mix old CSV formats.
   Existing merged CSVs are not the source of truth for sequence-only flow.
4. Class naming is API-level contract.
   Changing class key format later requires migration logic.

## Definition of Done

1. New sign can be added with one `uv` command (`add-sign`).
2. Retrain regenerates `label_map.json` and both model artifacts.
3. Inference shows correct word text for both mapped and unmapped `CUSTOM_*` classes.
4. Entire flow uses only `model/two_hand_sequence_classifier/*`.
